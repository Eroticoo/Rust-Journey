# Day 6 - 模块系统（mod/pub/use）与 HashMap

## 今天学了什么
- `mod` 把代码分模块组织
- `pub` 控制可见性（默认私有）
- `use` 缩短路径、提升可读性
- `HashMap` 做键值映射与遍历

## 心路历程
今天最大的变化是从“写功能”走向“组织代码”。
我开始意识到，项目变大后，模块边界和可见性设计比单个函数更重要。

`pub` 的门禁模型非常直观：不开门就只能内部用；开门后外部才能访问。
`use` 则让我体会到路径管理的工程价值——不是炫技，而是降低认知负担。

HashMap 这部分让我建立了“映射”思维：
它不是数据库，不追求顺序，而是通过键快速定位值。
