# Day 1 - 所有权、借用、match、Option

## 今天学了什么
- 所有权（Ownership）转移规则
- 借用（`&T` / `&mut T`）与借用冲突
- `match` 模式匹配（覆盖所有分支）
- `Option`（`Some/None`）替代 null

## 心路历程
今天最大突破是理解了 Rust 的“严格”不是刁难，而是提前避免未来崩溃。
一开始对“所有权转移后变量失效”很不习惯，但在借用和借用冲突例子里逐渐体会到：
**Rust 是在编译期强制你把边界说清楚。**

我对细节（如 `String::from`、`::`、`{}`）进行了追问，没有跳过不懂点。
这让我从“能跑代码”升级到“知道为什么这样写”。
