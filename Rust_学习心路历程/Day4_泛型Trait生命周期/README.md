# Day 4 - 泛型、trait、生命周期

## 今天学了什么
- 泛型（`<T>`）与 trait bound（如 `T: PartialOrd`）
- trait 作为“能力证书”的设计思路
- 生命周期（`'a`）用于约束借用的有效期

## 心路历程
今天难度明显提升，进入 Rust 进阶核心。
泛型让我看到 Rust 的抽象能力：同一套逻辑可以安全复用到多种类型；
trait 让我理解了“先定义能力，再给不同类型实现，最后统一调用”的工程化模式。

生命周期一开始最抽象，但通过“借用必须在有效时间内使用”的时间轴思路，
我逐渐接受它不是语法负担，而是 Rust 用来兜底内存安全的关键机制。

今天最大的收获是：
**Rust 的高级特性依然服务于同一个目标——把不确定性前置到编译期。**
