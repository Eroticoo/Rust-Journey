# Day 2 - 结构体、方法、Result、?

## 今天学了什么
- `struct` 组织数据
- `impl` 给结构体添加行为
- `Result<Ok, Err>` 错误处理
- `?` 进行错误传播

## 心路历程
今天进入了“工程化思维”阶段：不仅写逻辑，还要设计数据结构和错误路径。
对 `struct` + `impl` 的分工理解很快，能把抽象落在“手机对象”这种具体模型上。

中间有一次 `bool` 计算粗心，但概念本身是清楚的，修正后更稳。
对 `Result` 与 `?` 的理解让我感受到 Rust 的风格：
**失败不是例外，而是必须显式处理的常态。**
