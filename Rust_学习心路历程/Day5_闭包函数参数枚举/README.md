# Day 5 - 闭包、函数作为参数、枚举

## 今天学了什么
- 闭包（Closure）与捕获外部变量
- 函数/闭包作为参数传递
- 枚举（enum）表达状态分支并配合 `match`

## 心路历程
今天开始把 Rust 的“表达能力”串起来用了。
闭包让我感受到：逻辑不一定要提前定义成函数，
可以在使用位置快速组合，尤其是捕获外部变量这一点非常实用。

把函数当参数传递后，我明显体会到“把策略交给调用者”的设计思想，
代码灵活性提升了一个层级。

枚举这一节是今天最关键的连接点：
我不再把 `Option` 和 `Result` 当成特殊语法糖，
而是理解为 enum 在标准库里的具体实践。

今天的收获可以总结为：
**Rust 不只是安全，还在引导我写出更可组合、可维护的代码结构。**
